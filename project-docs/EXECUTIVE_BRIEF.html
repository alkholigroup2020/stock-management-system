Executive Brief

Stock Management System

Multi-location inventory, procurement, and period-close. Replacing the legacy Excel workbook.

A web application that gives every site a single, controlled way to receive stock, record consumption, move items between locations, and close the month. Built for speed on the floor, oversight for supervisors, and a clean audit trail for management.

What this document contains

  1. Main features: a one-page snapshot per capability area.
  2. Roles & responsibilities: who does what in the system.
  3. A testing scenario: a complete end-to-end test from initial setup to period close.

1. Main features at a glance

Capability What it does
Multi-Location Management Create unlimited locations (Kitchen, Store, Central, Warehouse). Each location has its own stock, WAC, and transaction history. Users can be assigned to one or many locations.
Items & Pricing Global item master with categories and units (KG, EA, LTR, BOX, CASE, PACK). Period prices lock when the period opens, ensuring consistent costing all month.
Bulk Item Import Import hundreds of items from Excel/CSV with validation and a downloadable template. Columns: Code, Name, Unit, Category, Subcategory.
Procurement (PRF → PO) Operators raise Purchase Requisitions. Supervisors approve. Procurement Specialists turn approved PRFs into POs. POs are emailed automatically to suppliers with VAT (15%) calculated.
Deliveries Receive stock against an open PO. Quantities pre-filled. Over-deliveries held for supervisor approval. Posting recalculates Weighted Average Cost (WAC). Price variance auto-creates an NCR with zero tolerance.
Issues (Stock Usage) Record consumption to a cost centre (FOOD / CLEAN / OTHER). System blocks any issue that would cause negative stock.
Transfers Move stock between sites with mandatory supervisor approval. Stock moves atomically at the source location's WAC.
NCRs (Non-Conformance) Price-variance NCRs auto-created on delivery. Manual NCRs for damages, short shipments, quality issues. Automatic email to Finance, Procurement, and the Supplier. Resolution tracks financial impact (CREDIT, LOSS, NONE) which feeds reconciliation.
Stock Now Real-time inventory view per location and consolidated. Search, category filter, low-stock filter, CSV export.
POB (Personnel On Board) Daily headcount per location (mandays + visitors meals) with auto-save. Powers cost-per-person reporting.
Reconciliations Per-location and consolidated views. Automated formula: Opening + Receipts + Transfers In − Transfers Out − Issues. NCR credits/losses included.
Period Close All locations must be marked READY. Admin requests close, then approves it. Snapshots are taken per location and opening balances roll forward to the next period.
Reports Stock Now, Deliveries, Issues, and Reconciliation reports. CSV export for further analysis.
Notifications Configurable Finance and Procurement email lists. Automatic emails for PRF approval, PO creation, PO closure, over-delivery approval, and NCR creation.
PWA & Offline Awareness Installable on any device. Disables write actions when offline so users never lose work.
Audit Trail Every transaction records who created, posted, and approved it, with timestamps. Closed periods are immutable.

2. Roles & responsibilities

The system has four roles, each scoped to what that person actually needs. Permissions are enforced both at the screen and at the API.

Role Location Access Can do
Operator Assigned locations only Create & submit PRFs · Post Deliveries against POs · Post Issues · Enter daily POB · Create transfer requests · View stock · Create manual NCRs
Supervisor All locations Everything an Operator can do, plus: approve / reject PRFs · approve over-deliveries · approve transfers · close POs · manage reconciliations · resolve NCRs with financial impact · view consolidated reports
Procurement Specialist All locations (read-only context) See approved PRFs · create & edit Purchase Orders · resend PO emails to suppliers (focused role: no access to deliveries, stock, or other modules)
Admin All locations Full system access · manage users · manage locations · manage items · manage suppliers · set period prices · execute period close · configure notification recipients · resend failed NCR notifications
Approval workflow at a glance. PRFs are approved by Supervisors. Transfers are approved by Supervisors. Over-deliveries are approved by Supervisors. PO closures with unfulfilled lines need a Supervisor or Admin and a closure reason. Period close is requested and approved by an Admin. Everything else posts immediately with a full audit trail.

3. A Testing Scenario

The following scenario takes you from a clean install to a fully closed period, touching every major workflow on the way. It can be completed in a single sitting and is designed for a live demo. Each step calls out which role performs the action and what success looks like.

Goal: Set up one location, raise a PRF, convert it to a PO, receive the delivery, consume some stock, transfer between sites, raise and resolve an NCR, then close the period, all in roughly 30 to 45 minutes.

Phase A: Initial setup (Admin)

1

Create the first locations

Admin
  1. Open Locations in the left menu.
  2. Click New Location. Create at least two locations, e.g. "Central Warehouse" (type CENTRAL) and "Site Kitchen 1" (type KITCHEN). Both will be needed for the transfer step later.
Verify: Both locations appear in the location switcher in the top bar.
2

Create user accounts for each role

Admin
  1. Open Users → Create User. Create one user per role: an Operator, a Supervisor, a Procurement Specialist, and (if needed) a second Admin.
  2. For the Operator, set their Default Location to the Kitchen.
  3. After creation, open the Operator's profile and on the Edit page assign them to both locations under Locations (this enables transfers).
Verify: Each new user can log in and only sees the menus relevant to their role.
3

Create suppliers

Admin
  1. Open Suppliers → New Supplier. Create one supplier with: Code, Name, Contact Person, at least one Email (this address will receive PO and NCR emails), Phone/Mobile, VAT Reg No., Address.
Verify: Supplier appears in the suppliers list.
4

Create items (manual or bulk import)

Admin
  1. Open Items. Either:
    • Click New Item and create 3–5 items by hand (Code, Name, Unit, Category, optional Sub-Category), or
    • Click Import Items → Download Template. Fill the five columns (Code, Name, Unit, Category, Subcategory) and upload the file. Review the validation preview, then confirm.
Verify: Items appear in the items list with the correct units and categories.
5

Create the first accounting period

Admin
  1. Open Periods → New Period. Set name, start date, end date.
  2. The period is created in DRAFT.
Verify: Period appears in the list with status DRAFT.
6

Set period prices and open the period

Admin
  1. Open the DRAFT period. Click Manage Prices.
  2. Enter a price for each item (or, if you have a previous period, click Copy from previous period).
  3. Click Save Prices, then Open Period. Prices are now locked.
Verify: Period status changes to OPEN. The period indicator in the top bar shows OPEN.
7

Configure notification recipients

Admin
  1. Open Settings → Notifications.
  2. Add the Finance Team and Procurement Team email addresses. Save.
Verify: Both email lists are saved. (NCR emails will be sent here in Phase D.)

Phase B: Procurement cycle

8

Operator enters daily POB

Operator
  1. Log in as the Operator. Confirm the active location is the Kitchen.
  2. Open POB. Enter Mandays and Visitors Meals for today and a few prior days. Entries auto-save on blur.
Verify: Total Mandays in the summary card increases as you fill in days.
9

Operator creates a PRF

Operator
  1. Open Orders → New PRF.
  2. Choose Type (e.g., NORMAL) and Category (e.g., FOOD).
  3. Add 2–3 line items with quantities and estimated prices. Add a justification note.
  4. Click Submit for Approval.
Verify: PRF status moves to PENDING. PRF number follows PRF-{LOCATION}-{DD}-{Mon}-{YYYY}-{NN}.
10

Supervisor approves the PRF

Supervisor
  1. Log in as Supervisor. Open Orders, filter by PENDING.
  2. Open the PRF, review the lines, click Approve.
Verify: PRF status moves to APPROVED. The Procurement Specialist receives an email notification.
11

Procurement Specialist creates the PO

Procurement
  1. Log in as Procurement Specialist. Open the approved PRF.
  2. Click Create PO. Choose the supplier created in step 3.
  3. Confirm or adjust quantities and unit prices. VAT is auto-calculated at 15%.
  4. Click Save.
Verify: PO is created with status OPEN, PO number follows PO-{DD}-{Mon}-{YYYY}-{NNN}, and the supplier receives the PO email automatically.

Phase C: Receiving and using stock

12

Operator posts the delivery

Operator
  1. Log in as Operator. Open Deliveries → New Delivery.
  2. Select the PO created in step 11. Supplier and lines auto-populate.
  3. Enter Invoice Number and Delivery Date.
  4. For at least one line, enter a unit price different from the period price (this triggers a price variance for the demo).
  5. Click Post Delivery.
Verify:
  • Delivery is POSTED; stock at the Kitchen increases.
  • WAC for each item is recalculated.
  • An NCR with type PRICE_VARIANCE is auto-created and visible in the NCR menu.
  • Notifications are sent to Finance, Procurement, and the Supplier.
13

(Optional) Demonstrate over-delivery approval

Operator Supervisor
  1. Operator: create a second delivery against the same PO with a quantity exceeding the remaining qty on a line. Save as draft and click Send for Approval.
  2. Supervisor: open Deliveries, filter by Pending Approval, open the delivery, review the over-delivery, then either Approve (delivery posts) or Reject with a reason (delivery is locked permanently).
Verify: Delivery shows the correct outcome; on approval, stock updates and the operator receives an email.
14

Operator records an Issue (consumption)

Operator
  1. Open Issues → New Issue.
  2. Pick the issue date and the Cost Centre (FOOD / CLEAN / OTHER).
  3. Use Sync Stock to load all on-hand items, enter quantities to issue, and remove items you don't need.
  4. Click Create Issue.
Verify: Stock decreases at the Kitchen; the issue value is calculated at current WAC. Trying to issue more than on-hand is blocked with a clear error.

Phase D: Inter-location transfer and NCR resolution

15

Operator creates a transfer request

Operator
  1. Open Transfers → New Transfer.
  2. Source: Site Kitchen 1. Destination: Central Warehouse.
  3. Add 1–2 line items with reasonable quantities. Add a justification note.
  4. Click Create Transfer.
Verify: Transfer is created with status PENDING_APPROVAL.
16

Supervisor approves the transfer

Supervisor
  1. Log in as Supervisor. Open Transfers, filter by Pending Approval.
  2. Open the transfer, verify quantities and locations, click Approve.
Verify: Transfer status is COMPLETED. Stock decreased at the Kitchen and increased at the Central Warehouse atomically. Destination WAC adjusts if it had prior stock.
17

Supervisor resolves the auto-generated NCR

Supervisor
  1. Open NCR. Find the price-variance NCR from step 12.
  2. Open it, review the variance, set status to RESOLVED, choose a Financial Impact: CREDIT (money recovered), LOSS (absorbed), or NONE.
  3. Add a brief resolution note and save.
Verify: NCR status changes; the financial impact appears in the location's reconciliation as either NCR Credits or NCR Losses.

Phase E: Reconciliation and period close

18

Supervisor reviews reconciliations and marks each location ready

Supervisor
  1. Open Reconciliations. Switch through each location.
  2. Verify the formula values: Opening + Receipts + Transfers In − Transfers Out − Issues = Expected Closing.
  3. Enter any required adjustments (Back Charges, Credits, Condemnations) with notes.
  4. Confirm NCR Credits / Losses are reflected.
  5. Click Mark Ready for each location.
Verify: Each location's status changes to READY. The consolidated view shows all locations green.
19

Admin requests period close

Admin
  1. Log in as Admin. Open Period Close.
  2. Review the pre-close checklist (deliveries posted, issues posted, transfers completed, all locations READY).
  3. Click Close Period to request close.
Verify: Period status changes to PENDING_CLOSE. An approval record is created.
20

Admin approves the close

Admin
  1. Approve the period-close request.
  2. The system briefly enters status APPROVED, then takes closing snapshots per location and finalises to CLOSED.
  3. Opening balances roll forward to the next period (which can now be created in DRAFT).
Verify:
  • Period is CLOSED; no further postings allowed.
  • Closing snapshots stored per location.
  • You can now create the next period in DRAFT and use Copy from previous period to bring prices forward.
